# \# Product Proposal: SkillSynth AI

# 

# \### 1. Обоснование идеи

# Проект решает прикладную задачу автоматизации работы "методиста". В отличие от простого чата с GPT, система автономно проверяет ссылки на работоспособность и актуальность, используя внешние API, и сохраняет состояние обучения пользователя.

# 

# \### 2. Цель и метрики

# \*\*Цель:\*\* Сократить время на подбор качественных учебных материалов с часов до 1-2 минут.

# 

# \*\*Метрики успеха:\*\*

# \* \*\*Product:\*\* Количество кликов пользователя до получения финального плана (цель < 5).

# \* \*\*Agentic (Success Rate):\*\* Доля планов, в которых все 100% ссылок соответствуют теме и уровню сложности.

# \* \*\*Technical:\*\* Latency генерации плана (p95 < 30 сек).

# 

# \### 3. Сценарии использования и Edge-кейсы

# \* \*\*Сценарий:\*\* Пользователь с базой в Python хочет выучить LangGraph за 5 часов в неделю.

# \* \*\*Edge-кейс 1 (Empty Search):\*\* По теме нет бесплатных видео. Агент должен переключиться на поиск научной документации или статей.

# \* \*\*Edge-кейс 2 (Outdated Tech):\*\* Пользователь просит план по старой библиотеке. Агент должен предупредить об устаревании и предложить актуальный аналог.

# 

# \### 4. Архитектурный набросок

# \*\*Модули:\*\*

# \* \*\*Profiler:\*\* Извлекает сущности (уровень, время, цель) из диалога.

# \* \*\*Planner:\*\* Разбивает тему на логические модули (пререквизиты -> база -> практика).

# \* \*\*Researcher (Tool Calling):\*\* Вызывает Tavily API и YouTube API.

# \* \*\*Critic/Validator:\*\* LLM-модуль, проверяющий найденный контент на "инфомусор".

# 

# \*\*Технологии:\*\* LangGraph (оркестрация), SQLite (память состояний), Streamlit (интерфейс).

# 

# \*\*Data Flow:\*\*

# User Input -> \[Profiler] -> State Update -> \[Planner] -> \[Researcher Tools] -> \[Validator] -> \[Notion/Markdown Output].
