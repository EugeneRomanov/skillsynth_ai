# Governance: Risks & Security

### 1. Risk Register

| Риск | Вероятность / Влияние | Детекция | Защита | Остаточный риск |
| :--- | :--- | :--- | :--- | :--- |
| Галлюцинация ссылок | Высокая / Высокое | Автоматический HTTP-запрос (ping) | Инструмент-валидатор перед выдачей ответа | Низкий |
| Prompt Injection | Средняя / Среднее | Анализ паттернов в запросе | Использование системных промптов и Sandbox для Tool Calling | Минимальный |
| Устаревший контент | Средняя / Высокое | Проверка даты публикации через API | Фильтр в поисковом запросе (last 12-24 months) | Средний |

### 2. Работа с данными и безопасность
* **Персональные данные:** Агент не запрашивает e-mail или пароли. Используется только профиль интересов.
* **Безопасность выполнения:** Инструменты поиска работают в режиме "read-only". Выполнение стороннего кода (Python REPL) отключено для предотвращения атак.
* **Подтверждение действий:** Экспорт в Notion происходит только после финального подтверждения (Human-in-the-loop).
